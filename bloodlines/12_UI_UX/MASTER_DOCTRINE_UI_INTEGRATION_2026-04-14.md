# Bloodlines Master Doctrine UI Integration - 2026-04-14

## Core Rule

The doctrine does not ask the UI to simplify Bloodlines by removing depth.

It asks the UI to make deep state readable.

## Required Readability

The player must be able to understand:

- water condition
- food condition
- population strain
- loyalty and legitimacy condition
- bloodline state
- doctrine posture
- defensive readiness
- whether the realm is truly healthy or only temporarily militarized

## UI Implication

The correct solution is:

- stronger hierarchy
- better state surfacing
- clearer iconography
- more legible world overlays
- clearer bloodline and realm-condition summaries

The incorrect solution is:

- cutting systems away because they are hard to show

## Practical Standard

If a player frequently asks "what am I looking at?" or cannot understand why the realm is failing, the UI has failed the doctrine even if the mechanics are technically present.
