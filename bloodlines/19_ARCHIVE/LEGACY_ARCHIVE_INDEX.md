# Legacy Archive Index — Pre-Project Files

**Archived:** 2026-03-19 (Session 9 cleanup)
**Source:** `D:\ProjectsHome\bloodlines game\` (legacy working folder, pre-project-structure)

These three files represent the original design work that seeded the Bloodlines project before the formal archive structure was created. They predate Sessions 1-9 and contain the foundational design intent. All content has been superseded, expanded, or formalized in the main project files, but these originals are preserved intact per archival rules.

---

## Files Archived

### LEGACY_AI_Copy_Prompt_pre-project.txt (~78KB)
**Original filename:** `AI Copy Prompt.txt`
**Content:** Project Bible + AI Memory Injection Prompt. A combined document designed to bootstrap AI sessions with full project context. Likely produced in early design work before the formal session ingestion pipeline was established. Overlaps substantially with `bible.txt`. Contains the original framing of the game's core identity: dynasty, faith, sacrifice, and conviction as interconnected pillars.

### LEGACY_bible_pre-project.txt (~63KB)
**Original filename:** `bible.txt`
**Content:** The original project Bible document. Contains same or similar content to the AI Copy Prompt file — the foundational game design document from the pre-structure era. This is the ancestor document of `18_EXPORTS/BLOODLINES_COMPLETE_DESIGN_BIBLE.md`, which is now 2,813 lines and fully supersedes this file.

### LEGACY_master_prompt_bloodline_details_pre-project.txt (~30KB)
**Original filename:** `master prompt - bloodline details.txt`
**Content:** The original canonical design memory document that seeded the current `01_CANON/BLOODLINES_MASTER_MEMORY.md`. This is the most historically significant of the three files. Notable unique content it contains:

**Legitimacy and Perception Layer** — Military success, wealth, cultural dominance, and religious infrastructure can make harsh actions seem justified to populations. Legitimacy affects loyalty and sabotage. (Referenced in system files; not yet a standalone formal system.)

**Recruitment Slider System** — Three slider components: (1) family obligation, (2) paid professional soldiers, (3) faith militancy volunteers. Sons and daughters produce different unit profiles — daughters have negative attack buffer but positive healing and sustainment buffer. Sons provide more conventional offensive profiles.

**Covert Action and Assassination** — The dynasty can conduct covert assassination operations targeting enemy bloodline members. Separate from open warfare.

**Captured Bloodline Member Options** — Four handling options: (1) enslaved as workers at home, (2) taken as wives/husbands, (3) killed outright, (4) ransomed back. Each produces different diplomatic, conviction, and population outcomes.

**Blood and Marriage Politics** — Detailed hybrid heir mechanics: repeated cross-dynasty marriages produce children with complex inheritance and legitimacy dynamics. Children of mixed bloodlines can branch off as new dynasties. Late-game dynastic web creation through accumulated marriages.

**Original Eight House Names (pre-canonicalization):** Trueborn, Highborne, Ironmark, Hartborne (became Hartvale), Whitehall, Westgrave (became Westland), Goldgrave, Oldcrest. Stonehelm was added as the ninth house in later sessions.

**Original Four Faith Names (exact canonical form):**
- The Old Light (Covenant of the First Flame)
- The Blood Dominion (The Red Covenant)
- The Order (Covenant of the Sacred Mandate)
- The Wild (Covenant of the Thorned Elder Roots)

**Family Tree UI Mandate** — "A family tree interface is a critical part of the game and must be a key area the player can always access. The dynasty tree must be treated as a core system and a core UI pillar, not a minor screen." This mandate is preserved and formalized in DYNASTIC_SYSTEM.md.
