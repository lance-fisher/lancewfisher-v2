# 2026-04-17 Unity Graphics Infrastructure Foundation

## Goal

Replace the debug primitive presentation bridge as the first-class runtime
rendering path with a real URP + per-instance-tint rendering layer that
commercial art can plug into without system rewrites. Add faction-color
shading, silhouette legibility, and low-poly procedural placeholder meshes
so unit CLASSES and building CLASSES are visually distinct immediately,
while final faction-identity art decisions stay human-owned and future.

This slice is **infrastructure, not a shipping visual pass**. No commercial
textures, no hand-authored animations, no VFX, no character portraits, no
terrain art, no lighting polish. Those land later with human art direction.

## Work Completed

### Shader
- Added `unity/Assets/_Bloodlines/Shaders/BloodlinesFactionColor.shader`:
  hand-written URP-compatible lit shader with:
  - base albedo + texture
  - per-instance `_FactionColor` with configurable strength for faction
    silhouette readability
  - rim-light for RTS-zoom silhouette pop (`_RimColor`, `_RimStrength`,
    `_RimPower`)
  - selection highlight uniform (`_SelectionHighlight`) blending toward
    `_SelectionColor` for Play Mode selection pop
  - `multi_compile _ DOTS_INSTANCING_ON` path so Entities.Graphics batch
    rendering can override per-instance properties without material
    cloning. Falls back to `Universal Render Pipeline/Lit` if the custom
    pass can't compile.
  - The brief requested a Shader Graph file (`.shadergraph`) but those
    assets are generated by the Unity editor UI, not authored as source.
    A hand-written `.shader` gives the same capability surface and is
    git-friendly infrastructure.

### Visual definitions (data layer)
- Added `unity/Assets/_Bloodlines/Code/Definitions/UnitVisualDefinition.cs`:
  ScriptableObject with `id`, `mesh`, `material`, `silhouetteScale`,
  `verticalOffset`, `castsShadows`. Keyed by the same string id as the
  existing gameplay `UnitDefinition`.
- Added `unity/Assets/_Bloodlines/Code/Definitions/BuildingVisualDefinition.cs`:
  parallel shape for buildings.
- **Additive only** — neither `UnitDefinition` nor `BuildingDefinition`
  was touched, and `JsonContentImporter` is unchanged.

### Faction tint
- Added `unity/Assets/_Bloodlines/Code/Components/FactionTintComponent.cs`:
  per-entity `float4` tint fed into the shader via
  `MaterialPropertyBlock`.
- Added `unity/Assets/_Bloodlines/Code/Rendering/FactionTintPalette.cs`:
  static color table covering player/Ironmark, enemy/Stonehelm, Tribes,
  Neutral/Trueborn City, Goldgrave, Hartvale, Westland, Whitehall,
  Oldcrest, Highborne, Trueborn, plus a deterministic hash fallback
  for any faction id not yet in the table.
- Added `unity/Assets/_Bloodlines/Code/Rendering/FactionTintAttachSystem.cs`:
  `ISystem` in `InitializationSystemGroup`. Attaches a
  `FactionTintComponent` to any unit or building with a
  `FactionComponent` but no tint yet. Uses
  `EndSimulationEntityCommandBufferSystem` for structural adds.

### Runtime rendering path
- Added `unity/Assets/_Bloodlines/Code/Rendering/BloodlinesVisualCatalog.cs`:
  editor-backed registry that loads every `UnitVisualDefinition` and
  `BuildingVisualDefinition` via `AssetDatabase.FindAssets` on first
  access, keyed by `id`. Public `TryGetUnitVisual` / `TryGetBuildingVisual`
  surface for system consumers. `ForceRefresh` exists for tooling.
- Added `unity/Assets/_Bloodlines/Code/Rendering/BloodlinesVisualPresentationBridge.cs`:
  `MonoBehaviour` that walks the ECS world every `LateUpdate`, looks up
  each unit/building's visual definition, creates or updates a
  `MeshFilter` + `MeshRenderer` proxy child GameObject, applies the
  faction tint + selection highlight via a reusable
  `MaterialPropertyBlock`, and cleans up stale proxies for despawned
  entities. Exposes `TryGetVisualProxyMeshAndMaterial(entity, ...)` and
  `HasVisualProxy(entity)` for governed validation.

### Placeholder asset generation
- Added `unity/Assets/_Bloodlines/Code/Editor/BloodlinesPlaceholderMeshGenerator.cs`:
  editor-only menu `Bloodlines -> Graphics -> Generate Placeholder Meshes`.
  Generates procedural low-poly meshes covering:
  - 18 unit silhouettes: villager, worker, militia, swordsman, bowman,
    axeman, light_cavalry, scout_rider, verdant_warden, commander,
    tribal_raider, siege_engineer, supply_wagon, battering_ram,
    siege_tower, trebuchet, ballista, mantlet
  - 17 building silhouettes: command_hall, dwelling, farm, well, barracks,
    stable, siege_workshop, supply_camp, wayshrine, market, storehouse,
    granary, keep_tier_1, watch_tower, gatehouse, wall_segment,
    tribal_camp
  - One shared `Placeholder_FactionShared.mat` material bound to the
    `Bloodlines/FactionColor` shader
  - Paired `UnitVisualDefinition` / `BuildingVisualDefinition`
    ScriptableObjects under `Assets/_Bloodlines/Data/UnitVisualDefinitions/`
    and `Assets/_Bloodlines/Data/BuildingVisualDefinitions/`
  - Each silhouette is a distinct shape class (tapered post, broad
    shoulder, slender bow, axeman, cavalry, warden, commander, wagon,
    keep, cottage, low plot, well, barracks-long, workshop, shrine,
    silo, tower, gate, wall segment), readable at RTS zoom. **No faction
    identity is baked in** — faction color comes from the shader's
    per-instance tint.

### Governed validator
- Added `unity/Assets/_Bloodlines/Code/Editor/BloodlinesGraphicsRuntimeValidation.cs`:
  `InitializeOnLoad` + `[MenuItem]` + `RunBatch` entry. Opens the
  Bootstrap scene, attaches `BloodlinesVisualPresentationBridge` to the
  authoring anchor if missing, enters Play Mode, warms up, and asserts:
  - Every spawned unit with a `UnitVisualDefinition` has a live visual
    proxy with non-null mesh + material
  - Every spawned building with a `BuildingVisualDefinition` has a live
    visual proxy with non-null mesh + material
  - Every such entity carries a `FactionTintComponent`
  Writes the outcome to `artifacts/unity-graphics-runtime.log` and
  exits batch mode with the pass/fail status.
- Added `scripts/Invoke-BloodlinesUnityGraphicsRuntimeValidation.ps1`:
  batch-mode wrapper invoking
  `Bloodlines.EditorTools.BloodlinesGraphicsRuntimeValidation.RunBatch`,
  consistent with the other governed wrappers, intended for use via
  `scripts/Invoke-BloodlinesUnityWrapperWithLock.ps1 -Session "claude-graphics"`.

### Debug bridge behavior
- `BloodlinesDebugEntityPresentationBridge` was briefly modified to skip
  primitive proxies for entities claimed by the new visual bridge, but
  that reduced its child count below the bootstrap runtime smoke's
  `expectedProxyMinimum` and failed the governed gate. Reverted back to
  additive coexistence: both bridges render in parallel. The visual
  bridge lives on the authoring anchor GameObject while the debug bridge
  keeps its original child list, so governance counts remain unchanged.
  A later slice can add a serialized toggle on the debug bridge that
  disables its primitives when the visual bridge is active, without
  touching the smoke validator.

## Current Readiness

The Unity first-shell now carries a real rendering pipeline alongside the
debug primitive bridge:

- URP faction-color shader with rim-light silhouette pop and selection
  highlight
- procedural placeholder meshes for every canonical unit class and
  building class that currently ships in `data/units.json` and
  `data/buildings.json`
- ScriptableObject visual layer keyed to the same `id` values the
  gameplay layer uses, so swapping commercial art later is one-asset-at-
  a-time instead of a full rewrite
- per-entity faction tint applied via `MaterialPropertyBlock`, backed by
  a canonical faction color palette covering all nine founding houses
  plus tribes/neutral
- governed `Graphics Runtime Validation` wrapper asserting the pipeline
  actually delivers mesh + material + tint for every definition-backed
  entity in Play Mode

Commercial art direction (unit silhouettes, building silhouettes,
material authoring, portraits, VFX, terrain, lighting) remains human-
owned and out of scope. This slice is the seat for that future art.
