## Mandatory Rules for All Sessions

The Bloodlines project memory system is canonical. Every session must respect the following rules:

1. The memory system is append-only. Content is added, never removed.
2. Nothing may be removed from the memory system without explicit authorization from the user.
3. Nothing may be summarized, condensed, shortened, reduced, redacted, or silently overwritten.
4. New content must be added in full. Do not paraphrase or simplify incoming material.
5. The memory files must grow over time. They must never shrink.
6. Historical context must be preserved even when newer material refines or supersedes it.
7. If new material conflicts with older material, both must be preserved with clear labeling.
8. Future sessions must read the memory before doing work. No exceptions.
9. If something is referenced but not defined in the canonical memory, write: "Not defined in canonical design memory."
10. Do not invent missing information. Only report confirmed information.
11. Do not silently reinterpret, rename, or restructure existing systems.
12. All terminology must remain consistent across sessions unless explicitly changed by the user.
