The Bookmark Bot

A multi-agent system that turns the noise of an X bookmarks feed into something useful. Seven asyncio agents read, enrich, score, route, and (for the worthwhile ones) scaffold the bookmark into a real project under governance.

Stack
source: bookmark-bot/CLAUDE.md

Python and Ollama. Nothing else for the core pipeline. The web UI is plain HTML, CSS, and JavaScript with no build step. The HTTP server is asyncio.start_server on raw TCP. The Builder agent calls Ollama through urllib. Fewer dependencies, fewer ways to break.

Pipeline
source: src/agents/, EventBus fan-out

The Ingester normalizes incoming bookmarks. The Analyzer and Researcher run in parallel: one extracts ideas, one fetches the underlying URL. The Scorer waits for both to land before merging them, then assigns a route. The Matcher checks the result against the existing ProjectsHome registry. The Reporter persists everything. The Builder is a separate downstream consumer.

The 7 Agents
source: src/agents/*.py, all async, all listening on the bus

Each agent is a single-file, single-purpose actor that subscribes to event types and publishes results. No agent reaches into another agent's state. Coordination happens through typed events on a shared async bus.

4-Dimensional Scoring
source: src/scoring/ranker.py, weights from config.py

Every idea is scored on four independent dimensions, then weighted into a single composite score. The weights live in config and are tunable. The final score is what the Router uses to assign a route.

Each dimension is calculated by a dedicated scorer module. Relevance looks at keyword match, venture alignment, and content type. Novelty rewards rare keywords and rare idea types. Actionability looks for how-to signals and specificity. Strategic fit measures alignment with declared ventures.

Routes
source: src/routing/router.py, hard-coded thresholds

Composite score lands the idea in one of four lanes. Only BUILD_NOW ideas are eligible to be picked up by the Builder agent. Everything else is preserved for later review. Nothing is deleted.

Real Ideas From The Pipeline
source: state/ideas.jsonl, real bookmark output

A live sample of ideas the pipeline has actually produced. Each card shows the real composite confidence, the classified idea type, the inferred content type, the extracted tags, and any venture mappings the matcher found. Ideas the matcher tagged for Fisher Sovereign Systems are highlighted.

Builder Agent
source: src/agents/builder.py, src/building/

When a BUILD_NOW idea passes review, the Builder picks it up and turns it into a real, registered project. It scaffolds the directory, writes the standard files, registers the project in PROJECTS.json, claims a port from the registry, and creates the GitHub repo. Code generation happens through a local Ollama model.

Governance Hooks
source: src/building/governance.py

A new project is not just a folder. It is an entry in PROJECTS.json, a port reservation, a GitHub repository, a CLAUDE.md file at the root, an .env.example, and a .gitignore. The Builder is responsible for all of these in one transaction so a half-built project can never enter the registry.

What Gets Created
Files and entries written for every BUILD_NOW success.
  • PROJECTS.jsonv2 entry with port, status, owner, GitHub URL
  • CLAUDE.mdstack, structure, how to run, design notes
  • .gitignorelanguage-aware, includes secret patterns
  • .env.examplescaffold for credentials, never .env itself
  • README.mdsummary, install steps, link back to source idea
  • port-registry.jsonport claim handed off to project-hub
What Never Gets Created
Categorical exclusions enforced by the scaffolder.
  • .envnever written under any condition
  • credentialsno API keys, tokens, or wallet material
  • node_modulesnot scaffolded, even for JS projects
  • build artifactsno compiled output committed
  • half-built projectsif any step fails, the whole transaction rolls back
Web UI & Auth
source: src/web/, asyncio HTTP server, SSE, OAuth 2.0 PKCE

A vanilla single-page app served by an asyncio HTTP/1.1 server. State updates stream over Server-Sent Events so the dashboard reflects the bus in real time. Authentication is OAuth 2.0 PKCE against X with an optional bypass mode for development.

Server
No frameworks, no WSGI, no ASGI.
  • transportasyncio.start_server, raw TCP HTTP/1.1
  • streamingSSE for real-time bus events
  • staticSPA shell, app.js router, components/*.js views
  • apihandlers in src/web/api.py
  • port8099 (assigned in project-hub registry)
Auth
OAuth 2.0 PKCE against X.
  • flowPKCE, no client secret on disk
  • sessionin-process session map, rate limited
  • bypassunset X_OAUTH_CLIENT_ID for dev mode
  • scopebookmark.read only
