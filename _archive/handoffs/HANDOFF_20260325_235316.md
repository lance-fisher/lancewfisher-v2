### ROOT CAUSE FOUND: `.p-title` Light Theme Color Bug

**Problem:** All 16 project card `.p-title` elements showed `rgb(240, 235, 224)` (dark mode color) in light theme, even with `!important` overrides.

**Root cause:** CSS `transition: ... color 0.4s ease` on `.project-card .p-title` (and `.ftda-flagship .p-title.ftda-accent`) creates a `CSSTransition` when the theme switches. The transition gets stuck at `currentTime: 0` with `fill: "backwards"` for off-screen elements, permanently displaying the FROM value (dark mode color). CSS transitions override all cascade layers including inline `!important`.

**Fix applied:** Removed `color` from the `transition` property on:
- `deploy/styles.css` line 516: `.project-card .p-title` transition
- `deploy/styles.css` line 532: `.project-card .p-desc` transition (same issue)
- `deploy/ftda/ftda.css` line 54: `.ftda-flagship .p-title.ftda-accent` transition
- `deploy/styles.css` line 648-650: `.thumb-label` — removed color transition + converted hardcoded `rgba(240,235,224,...)` to `rgba(var(--fg), ...)`

**Verified:** After fix, all elements show correct light mode colors:
- All 16 `.p-title`: `rgba(26, 26, 28, 0.92)` (0 animations)
- `.p-desc`: `rgba(28, 26, 24, 0.8)` (0 animations)
- `.thumb-label`: `rgba(42, 42, 46, 0.5)` (0 animations)

**IMPORTANT NOTE:** On a fresh page load, the theme must be set AFTER CSS is loaded. Residual CSSTransition objects from old cached CSS can persist even after stylesheet swap. A clean page load with the fixed CSS will not create any stuck transitions. The `?v=` cache-bust parameter in index.html should be bumped when deploying (`?v=20260322`).

## Key Insight for Future Reference

CSS `transition: color` properties cause a stuck `CSSTransition` on off-screen elements when the theme switches via `data-theme` attribute change. The transition freezes at `currentTime: 0` with `fill: "backwards"`, permanently showing the old (pre-switch) color. This overrides ALL cascade layers including inline `!important`. The fix is to remove `color` from transition properties on any element that uses CSS custom properties for theming.
