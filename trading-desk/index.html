Architecture Paper-First

Autonomous Trading Desk

A multi-agent trading system. profit-desk is the brain that researches, decides, and risk-checks. master-trade-bot is the execution layer that talks to exchanges. Every strategy is paper-first and must clear a promotion gate before a single dollar of real capital is allocated.

profit-desk / master-trade-bot / trading-shared

Signal Flow brain → execution → market

All trading signals originate in profit-desk. Strategies emit candidate trades. The Risk Manager veto-gates them. Approved orders are dispatched over the event bus to master-trade-bot, which is the only component permitted to talk to exchanges. Dashboards are read-only consumers of the same bus.

Agent Architecture 6 agents / profit-desk

Six specialized agents run inside profit-desk. Each owns one slice of the trading lifecycle. They communicate over an in-process event bus with structured event types. No agent reaches across boundaries to mutate another agent's state.

Strategies 4 strategies / profit-desk/strategies

Each strategy is a self-contained module under profit-desk/strategies. They share a common base class and emit signals into the same risk pipeline. None of them are live. The two oldest are running in paper mode against real market data; the two newest are still in research and backtest.

Risk Parameters profit-desk/config.yaml

Hard limits enforced by the Risk Manager agent before any order leaves profit-desk. These are real values from the project's config file. Position sizing, drawdown caps, and exposure limits are checked on every signal before it can become an order.

Promotion Gate backtest → paper → live

No strategy goes live on a hunch. The promotion gate is a hard-coded checklist a strategy must satisfy across a sustained paper-trading window before it earns a live capital allocation. As of today, zero strategies have been promoted.

Tech Stack

Two repositories, two languages, one event bus. profit-desk is Python because the strategy and research work lives there. master-trade-bot is TypeScript because every exchange SDK that matters ships in JS.

Safety Rails non-negotiable

These are not aspirational. They are encoded in the code, the configs, and the project-level governance files. Any change to any of them requires explicit operator approval; the default for every bot, every config, every fresh checkout is the safest possible setting.
